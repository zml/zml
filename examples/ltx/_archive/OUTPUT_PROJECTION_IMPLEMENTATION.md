## Architecture

`OutputProjection` implements Python's `velocity_model._process_output`. It is applied independently to the video and audio streams after all 48 transformer blocks.

### Full chain

```
x [B, T, D]  +  embedded_timestep [B, D]  +  scale_shift_table [2, D]
  â†’ shift = scale_shift_table[0] + embedded_timestep  [B, D]
  â†’ scale = scale_shift_table[1] + embedded_timestep  [B, D]
  â†’ LayerNorm(x, eps=1e-6, elementwise_affine=False)
  â†’ normed * (1 + scale) + shift
  â†’ proj_out  [B, T, D_out=128]
```

### Python reference (`velocity_model._process_output`)

```python
def _process_output(self, scale_shift_table, norm_out, proj_out, x, embedded_timestep):
    scale_shift_values = (
        scale_shift_table[None, None].to(dtype=x.dtype) + embedded_timestep[:, :, None]
    )
    shift, scale = scale_shift_values[:, :, 0], scale_shift_values[:, :, 1]
    x = norm_out(x)             # LayerNorm(eps=1e-6, elementwise_affine=False) â€” no learned params
    x = x * (1 + scale) + shift
    x = proj_out(x)
    return x
```

Key points:
- `norm_out` is `LayerNorm(elementwise_affine=False)` â€” **no weight or bias in the checkpoint**
- `scale_shift_table` is stored as **f32** in checkpoint, cast to activation dtype at runtime
- `embedded_timestep` comes from `adaln_single.forward` (the value **after `linear_2`**, before the final `silu+linear_out`)

## Implementation

### `OutputProjection` struct in `model.zig`

```zig
pub const OutputProjection = struct {
    pub const Params = struct {
        scale_shift_table: Tensor, // [.n_ssv=2, .d] â€” f32 in checkpoint
        proj_out: zml.nn.Linear,   // contracts .d â†’ .d_out (e.g. 4096 â†’ 128)
    };

    /// x: [.b, .t, .d], embedded_timestep: [.b, .d_emb] â€” from adaln_single.forward
    /// Returns: [.b, .t, .d_out=128]
    pub fn forward(x_in: Tensor, embedded_timestep: Tensor, params: Params) Tensor {
        // shift/scale: sst_row[d] + emb[b, d] â†’ [.b, .d]
        const shift = sst_shift.broad(emb.shape()).add(emb);      // sst[0] + emb
        const scale = sst_scale.broad(emb.shape()).add(emb);      // sst[1] + emb
        // LayerNorm (no affine, eps=1e-6), then scale-shift, then project
        const normed = zml.nn.normalizeVariance(x, 1e-6);
        const modulated = normed
            .mul(scale.addConstant(1.0).broad(x.shape()))
            .add(shift.broad(x.shape()));
        return params.proj_out.forward(modulated);
    }
};
```

**`forwardOutputProjection`** free function (compilation entry point):
```zig
pub fn forwardOutputProjection(x: Tensor, embedded_timestep: Tensor, params: OutputProjection.Params) Tensor
```

**`LTXModel.Params`** â€” 2 new fields:
```zig

audio_norm_proj_out: OutputProjection.Params,  // audio
```

Both wired in `initParams` and `unloadBuffers`.

## Parity Verification

### Synthetic fixture

The initial fixture was generated by **manually reimplementing `_process_output`** using raw checkpoint tensors and random synthetic inputs.

This validated:
- âś… Zig math matches the Python equations
- âś… Correct checkpoint key mapping
- âś… Correct f32 cast for `scale_shift_table`
- âś… Correct `d_emb â†’ d` rename for `embedded_timestep` broadcasting

### Live-pipeline fixture

A second fixture was generated by hooking into an actual `TI2VidTwoStagesPipeline` forward pass (step 0, token-limited to 512), capturing the real `x_in`, `embedded_timestep`, and `output` values flowing through `_process_output` for both streams.

- âś… Validates against actual mid-model activations from a real denoising step
- âś… Both video and audio streams compared end-to-end
